LLM Output Arbitration System

Three specialized critics on three different models evaluate any LLM output in parallel; an adjudicator resolves their disagreements into one confidence-scored verdict.

Architecture

How a request moves through the system.

1
Three critics, three models

Factual accuracy (GPT-4o slot), logical consistency (Claude slot), completeness (local Llama slot). Different models on purpose: shared weights share blind spots.

2
Structured critiques

Every critic returns the same Pydantic shape: score 1 to 5, issues with quote, problem, and severity, plus its own confidence.

3
Parallel fan-out

Critics run concurrently with graceful degradation; a failed critic lowers confidence for its dimension instead of killing the verdict.

4
Disagreement detection

Score spreads, lone findings, and severity gaps are flagged; they are the most valuable signal in the system.

5
Evidence-based adjudication

Critical findings stand on their quoted evidence; uncorroborated minor flags are dismissed with reasoning. All-clean reports short-circuit to a high-confidence pass.

How to use

Runs fully offline with a deterministic provider; add API keys to switch to live models.

# offline: arbitrate four canonical cases
pip install -r requirements.txt
python demo.py

# serve the arbitration API
uvicorn src.arbitration.api:app --reload
curl -X POST localhost:8000/v1/arbitrate \
  -d '{"output": "...", "question": "..."}'
